## 문제상황

- 중앙집권화된 시스템의 신뢰 비용
  - 기존 시스템(은행, 정부)는 중앙 서버 관리자를 전적으로 신뢰해야 함
  - 관리자의 실수, 해킹 등으로 데이터가 훼손될 가능성
  - 단일 실패 지점(SPOF: single point of failure) 존재. 중앙 서버 다운 시 서비스 전체 마비
- 데이터의 비가시성, 불투명성
  - 사용자는 자신의 데이터가 DB 내부에서 어떻게 처리되는지 확인할 방법이 없음
- 비효율적인 중개 프로세스
  - 해외 송금, 부동산 거래 등에서 중개 기관에 의해 높은 수수료, 긴 처리 시간

## 개념

- 분산 원장 기술 (DLT: distributed Ledger Technology)
  - 네트워크에 참여하는 모든 노드가 동일한 장부를 복사해 나누어 가짐
- Block + Chain
  - 데이터를 담은 블록을 해시 함수를 이용해 체인처럼 연결
  - 이전 블록의 해시값을 현재 블록이 갖고 있음 -> 과거 데이터 하나가 변조된다면 그 이후 모든 블록의 해시값이 깨짐 (데이터 무결성 보장)

## 핵심 기술 및 활용

- 해시 함수 (SHA-256): 단방향 함수. 입력값이 조금만 달라져도 결과값이 완전히 바뀜. 데이터 위변조 탐지 핵심
- 공개키 암호화 + 전자서명
  - 개인키로 서명 -> 소유권 증명
  - 공개키로 서명 검증 -> 신원 확인
- 합의 알고리즘
  - PoW(Proof of Work): 복잡한 연산 문제를 해결해 블록 생성 권한 획득 (비트코인). 보안성 good, 에너지 소비 큼
  - PoS(Proof of Stake): 지분증명. 지분을 많이 가진 노드가 블록을 검증 (이더리움 2.0). 에너지 효율적. 속도 빠름
  - PBFT(Practical Byzantine Fault Tolerance): 프라이빗 블록체인에서 사용. 소수의 지정된 노드 간 합의
- 스마트 컨트랙트
  - 블록체인 위에 배포되는 프로그램 코드. 특정 조건 만족 시 계약 내용이 자동으로 이행됨
    - ex) Solidity(이더리움 언어), EVM 상에서 바이트코드로 실행됨

# 활용

- 스마트 컨트랙트: DeFi(Decentralized Finance: 탈금융화 금융, 중개인 없음)의 자동 이자 지급, 보험금 자동 청구, NFT 발행 로직
- NFT(Non-Fungible Token): 디지털 자산의 소유권 및 거래 이력 영구 박제
- DID(Decentralized Identity: 탈중앙화 신원증명): 데이터 주권 회복. 개개인이 자신의 정보에 완전한 통제권을 가짐. 나를 인증하는데 왜 제 3자의 기관을 통해 해야 함? 내 신원 정보를 단말기에 저장하고, 필요할 때에만 제출 (백신 패스)
- 공급망 관리 (SCM: supply chain management): 원자재~최종 소비자까지 유통 과정의 투명한 추적

## 단점 및 극복방안

1. 블록체인 트릴레마

- 확장성, 보안성, 탈중앙화 세 가지를 동시에 달성하기 어려움
- 사용자가 몰리면 TPS가 급락, 수수료가 폭등
- 극복 방안
  - Layer 2 (Rollup): 거래는 Layer 2에서 처리, 결과값은 메인넷(Layer 1)에 기록
    - Optimistic Rollup, ZK-Rollup
  - Sharding: 데이터베이스의 파티셔닝, 샤딩처럼 네트워크를 여러 조각으로 나누어 병렬 처리

2. 오라클 문제

- 블록체인(On-chain)은 외부 세상(Off-chain)의 데이터를 스스로 가져올 수 없음. (날씨, 주식가격, 등등..)
- 외부 데이터를 내부로 가져오는 과정에서 신뢰성 훼손 가능성
- 극복방안
  - 탈중앙화 오라클 (Chainlink): 여러 개의 검증된 노드가 외부 데이터를 가져와 중간값을 산출해 입력

3. 프라이버시 침해
   - 퍼블릭 블록체인은 모든 거래 내역이 공개 -> 기업의 기밀, 개인의 자산 규모 노출

- 극복 방안
  - 영지식 증명 (Zero-knowledge Proof): 내용을 공개하지 않아도 해당 내용이 참임을 증명하는 수학적 알고리즘 도입.
  - (Zcash, Mina Protocol)

## 최신 동향

- RWA (Real world Assets) & STO(토큰 증권)
  - 부동산, 채권, 금 등 실물자산을 블록체인 기반 토큰으로 연동
  - 블랙록 등 글로벌자산운용사의 RWA 펀드 출시
  - 국내 조각투자 가이드라인 마련
- 계정 추상화 (Account Abstraction)

  - 문제: 기존 지갑은 개인키 분실시 복구 불가, 매번 가스비 필요
  - 개선: 스마트 컨트랙트 지갑을 통해 소셜 로그인 복구.

- 인터체인 및 모듈러 블록체인

  - 인터체인: 서로 다른 블록체인 (비트 - 이더) 간 데이터를 자유롭게 이동
  - 모듈러: 블록체인의 기능(실행, 합의, 데이터 가용성)을 여러 레이어로 쪼개서 각자 잘하는 체인끼리 조립해 사용

- CBDC(Central Bank Digital Currency)
  - 각국 중앙은행이 발행하는 디지털 화폐. 블록체인 기술을 활용하되, 중앙 통제가 가능한 형태

# 스테이블 코인

## 문제상황

- 기존 암호화폐의 극심한 변동성
- 법정 화폐와 블록체인 간 연결 불편
- DeFi 내에서 달러 가치를 가진 자산이 필요

## 개념

- 법정 화폐나 실물 자산에 가치를 고정하여 가격 변동성을 최소화한 암호화폐
- 블록체인의 장점(투명성, 보안, 빠른 전송) + 법정화폐의 장점(가격 안정성) 결합
- 1코인 == 1달러의 가치를 유지하도록

## 기술적 원리 및 유형

- 법정화폐 담보형
  - 발행사가 은행 계좌에 실제 달러를 예치, 그 금액만큼의 코인을 1대1로 매칭하여 발행
  - 가장 직관적, 단 발행사를 신뢰해야 함
  - USDT, USDC
  - 리스크: 외부 회계 감사 필수
- 암호화폐 담보형
  - 중앙 기관 없이 스마트 컨트랙트를 통해 이더리움 등 다른 코인을 담보로 맡기고 스테이블 코인을 빌림
  - 과담보: 담보 자산의 가격 변동을 대비해 더 많은 금액을 맡김 (150달러 어치 이더리움을 맡기고 100달러어치 코인 발행)
  - 완전한 탈중앙화. 담보 가치가 폭락하면 스마트 컨트랙트가 담보를 강제 청산하여 가치 유지
  - DAI
- 알고리즘 기반형
  - 담보 없이 알고리즘이 시장의 공급과 수요를 조절하여 1달러를 맞춤
  - 가격이 1달러보다 높으면 코인을 더 찍어냄 (Mint)
  - 낮으면 코인을 소가 (Burn)
  - 효율성 good, 신뢰가 깨지면 죽음의 소용돌이 발생 가능 (테라-루나 사태)
  - USDD

## 도입 의의

- 블록체인 생태계의 기축통화
  - 비트를 팔고 현금화 대신 스테이블코인으로 바꿔놓으면 자산을 블록체인 위에 둘 수 있음
- 국경 없는 송금, 결제 효율성
  - SWIFT 망을 거치지 않음
  - 중개사 거치지 않음
- 프로그래밍 가능한 돈
  - 스마트 컨트랙트와 결합하여 조건부 자동 결제, 실시간 급여 지급 등 다양한 파생상품 개발 가능

## 활용 방안

- DeFi의 핵심 연료
  - DEX(탈중앙화 거래소)에서 거래 쌍의 기준 통화 역할
- 국제 송금 및 원조
  - 인플레이션이 심한 국가에 달러 대체재로 활용, 자산 가치 보존
- 기업 간 결제

## 리스크

- 디페깅: 시장 충격, 신뢰 상실로 1달러 가치가 깨지는 현상
- 규제 불확실성
  - 중앙은행의 규제 움직임 - 기존 통화 정책 교란 가능성
- 중앙화 이슈
  - USDT, USDC같은 주류 코인은 발행사가 계좌를 동결할 수 있는 권한이 있음 -> 검열 저항성 위배 논란.
