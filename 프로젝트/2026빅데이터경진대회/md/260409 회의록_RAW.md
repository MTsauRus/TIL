## 이정태

대안 아이디어 2: "24시간 살아있는 서울: '비욘드 조닝(Beyond Zoning)' 도입을 위한 융복합 최적 입지 타겟팅"
이 주제는 2040 서울플랜의 또 다른 핵심인 '도시계획 대전환(Beyond Zoning Seoul)'을 다룹니다. 주거/상업/공업 등으로 엄격하게 분리된 과거의 용도지역제(Zoning)가 가진 한계를 빅데이터로 증명하고 복합개발이 필요한 곳을 찾는 고도화된 주제입니다.

문제인식 및 주제 적절성 (15점): 과거 산업화 초기에 만들어진 용도지역제는 미래의 융복합적인 토지이용수요에 선제적으로 대응하는 데 한계가 있습니다. 특정 지역이 낮에는 북적이지만 밤에는 유령도시가 되거나(도심 공동화), 밤에는 붐비지만 낮에는 죽은 공간(순수 주거지)이 되는 비효율성을 문제로 인식합니다.
서울 데이터 허브 및 데이터 활용 (15점): 용도지역 현황 데이터, 건축물 대장(용도별 면적), 시간대별 생활인구 데이터, 상권 매출 발생 시간대 데이터를 교차 분석합니다.
데이터 표현의 정확성 (20점): 각 블록(Block)이나 헥사곤 단위로 '주야간 인구 활력 편차 지수' 및 '건축물 용도 다양성 지수(Shannon's Entropy index 활용)'를 통계적으로 정확하게 산출합니다.
시각화의 전달력 및 창의성 (30점): 24시간 동안의 인구 및 상권 활성도 변화를 타임랩스(Time-lapse) 애니메이션 형태의 인터랙티브 대시보드로 구현합니다. 붉은색(상업/업무 편중), 푸른색(주거 편중)이 시간에 따라 어떻게 맥동(Pulsing)하는지 보여주고, 두 기능의 혼합이 필요한 회색 지대(Dead zone)를 시각적으로 찾아냅니다.
정책 기여도 (20점): 엄격한 용도지역 제약 때문에 토지의 효율적 이용에 한계를 겪고 있는 지역을 데이터로 증명하여, 서울시가 향후 '다기능 복합지역'이나 '용도지역 변경' 시범 사업지로 선정할 수 있는 과학적 근거 타겟 리스트를 제공합니다.

## 김성령

### 1. 끊어진 서울

「끊어진 서울」 — 시각화 5종 상세 설계
📐 핵심 분석 개념 한 줄

지상철도는 **선형 장벽(linear barrier)**이다. 시민이 이 장벽을 건너려면 **횡단점(crossing)**을 거쳐야 하며, 횡단점이 듬성듬성할수록 우회 보행거리가 폭증한다. 이 우회 페널티를 정량화·지도화한다.

시각화 1. 단절 지도 (Disconnection Heatmap)
무엇을 보여주는가
서울 전체 지도 위에서, 지상철도 양옆 500m 버퍼 안의 보행자가 반대편으로 가려면 평균 몇 m를 우회해야 하는가를 색으로 표현. 우회가 심한 구간일수록 빨갛게 빛나는 "출혈선"으로 보임.
정확히 어떻게 그려지는가 (단계별)
[1단계] 지상철도 선분(LineString)을 OSM에서 추출
└ railway=rail, bridge!=yes, tunnel!=yes 인 구간만

[2단계] 선분을 50m 간격으로 점(sample point)으로 분해
└ 약 2,000~3,000개 샘플점 생성

[3단계] 각 샘플점에서 양옆 200m 떨어진 곳에
가상의 "출발점 N"과 "목표점 S"를 만든다
(=철도 바로 옆에 사는 가상 시민)

[4단계] N→S로 가려면 보행 네트워크에서 최단경로를 계산
└ 직선거리는 400m
└ 실제 보행거리는? 600m? 1,500m? 3,000m?

[5단계] 우회비율 = 실제거리 / 직선거리 (1.0이면 단절 없음, 5.0이면 5배 돌아감)
└ 이 값을 샘플점에 색으로 매핑

[6단계] 샘플점들을 선으로 이으면 → 철도를 따라 흐르는 "출혈선"
결과물 이미지 (말로 묘사)
서울 지도 위에 검은 행정동 경계가 옅게 깔리고, 지상철도 노선만 굵게 빛난다. 단절 적은 구간(예: 신도림 일대 횡단 인프라 충실)은 연한 청록, 단절 심한 구간(예: 영등포-신길 일부, 중랑천 근접 구간 등)은 선홍색으로 빛남. 마치 동맥류처럼 빨간 점이 부풀어 있는 곳이 시각적 핵심.

- **적합성: ★★★★★** — 노인 친화 구역 지수는 곧바로 "복지시설 입지 선정", "저상버스 노선 재편", "무더위쉼터 증설 우선순위" 등 구체적 정책 액션 아이템으로 연결 가능. 서울시 어르신 정책과 직결되므로 정책 기여도가 매우 높음.
- **보강 전략**: 결론에서 "A동은 의료 점수는 상위 10%이나 교통 점수가 하위 20%이므로, 저상버스 노선 1개 추가 시 종합 지수가 X점 상승 예상" 같은 **구체적 시뮬레이션형 인사이트**를 제시하면 정책 실행력에서 차별화 가능.

### 1-3. 2025 우승작 대비 차별화 포인트

2025 우승작 "걸어서 15분, 문화에 닿는 서울"과의 구조적 유사성(보행일상권 + 접근성 지수화 + 행정동 단위 코로플레스)을 인정하되, 다음 세 가지로 차별화:

1. **대상 전환**: 문화시설 → 노인 생활 전반(의료·교통·복지·안전·기후 등 7대 영역). 분석 차원이 훨씬 다양하므로, 다차원 지표 간 **불균형(mismatch)** 분석이라는 새로운 시각 제공.
2. **수요-공급 갭 분석**: 단순 시설 커버리지가 아니라, "65세+ 인구 밀도 대비 시설 수"로 **수요 보정**을 적용. 시설이 많아도 노인 인구가 더 많으면 실질적으로 부족한 구역을 식별.
3. **취약 프로필 유형화**: 2025 우승작의 A/B/C/D 유형 분류를 발전시켜, K-means 등 클러스터링으로 동네별 "취약 유형"을 도출하고 유형별 맞춤 정책을 제안.

## 3. 데이터 전처리 계획 및 2차 산출물 정의

### 3-1. 전처리 파이프라인 개요

```
[원본 데이터 수집]
    → [행정동 코드 표준화]
    → [결측치 처리]
    → [파생 지표 산출]
    → [정규화]
    → [가중합산]
    → [최종 지수 테이블]
```

### 3-2. 세부 전처리 계획

**(1) 행정동 코드 표준화**

- 기준: 통계청 행정동 코드 (10자리). 서울시 행정동은 약 467개.
- 문제: 데이터 출처마다 행정동명·코드 버전 불일치 가능 (행정동 통폐합 반영 시점 차이).
- 해결: 통계청 최신 행정구역 코드를 기준 테이블로 삼고, 행정동명 → 행정동코드 매핑 사전을 만들어 모든 데이터를 이 기준으로 LEFT JOIN. 매칭 안 되는 동은 수작업 확인.

**(2) 좌표 → 행정동 변환**

- 약국, 전통시장, 쉼터 등 좌표(위경도)로 제공되는 데이터는 Python `geopandas`의 spatial join으로 행정동 경계 폴리곤에 매핑.
- 행정동 경계 shapefile: 통계청 통계지리정보서비스(SGIS)에서 다운로드.

**(3) 결측치 처리**

- 특정 행정동에 데이터가 없는 경우(예: 해당 동에 약국이 0개) → 결측이 아닌 실제 0으로 처리.
- 통계 데이터에서 "-" 또는 공백으로 표시된 값 → 0 또는 NaN으로 통일 후, 0으로 대체(시설 부재는 "없음"이 정보).

**(4) 인구 데이터와의 병합**

- 모든 시설 수 지표는 "절대 수"와 함께 "65세+ 인구 1,000명당 비율"을 산출하여 수요 보정.

### 3-3. 2차 파생 데이터 정의

| 파생 변수명              | 산출 수식                                        | 의미                                                      |
| ------------------------ | ------------------------------------------------ | --------------------------------------------------------- |
| `clinic_per_1k_senior`   | (내과+정형외과+한의원 수) / (65세+ 인구 / 1000)  | 노인 1,000명당 1차 진료기관 수. 높을수록 의료 접근성 양호 |
| `pharmacy_per_1k_senior` | 약국 수 / (65세+ 인구 / 1000)                    | 노인 1,000명당 약국 수                                    |
| `lowfloor_bus_density`   | 저상버스 정류장 수 / 행정동 면적(km²)            | 단위 면적당 저상버스 정류장 밀도                          |
| `subway_elevator_ratio`  | 엘리베이터 보유 역 수 / 행정동 내 전체 역 수     | 교통약자 편의시설 보급률                                  |
| `welfare_capacity_ratio` | (경로당+복지관+노인교실 정원 합) / 65세+ 인구    | 노인복지시설 수용 여력. 1 이상이면 수요 충족              |

| `market_density`         | (전통시장 + 슈퍼마켓 수) / 행정동 면적(km²)      | 생필품 구매처 물리적 밀도                                 |
| `cctv_density`           | CCTV 수 / 행정동 면적(km²)                       | 치안 인프라 밀도                                          |
| `senior_accident_rate`   | 노인 보행사고 건수 / 65세+ 인구 × 10,000         | 노인 만 명당 보행사고 발생률                              |
| `shelter_per_1k_senior`  | (무더위쉼터 + 한파쉼터) 수 / (65세+ 인구 / 1000) | 기후재난 대피 접근성                                      |
| `park_area_ratio`        | 행정동 내 공원 면적 / 행정동 전체 면적           | 녹지 환경 비율                                            |

## 4. 점수 산정 방식 및 최종 취약 구역 선정 방법

### 4-1. 개별 지표 정규화

**방법: Min-Max 정규화 (0~1 스케일)**

```
X_norm = (X - X_min) / (X_max - X_min)
```

- 모든 소분류 지표를 0(최악)~1(최선) 범위로 변환.
- **역방향 지표** (값이 높을수록 취약한 지표, 예: `solo_senior_ratio`, `senior_accident_rate`)는 반전:
  ```
  X_norm = 1 - (X - X_min) / (X_max - X_min)
  ```
- 이상치(outlier) 영향을 줄이기 위해 상·하위 1% 윈저화(winsorization) 적용 후 정규화.

### 4-2. 영역별 점수 산출

7대 영역별로 소속 소분류 지표의 **단순 평균**으로 영역 점수를 산출:

```
Score_의료 = mean(clinic_per_1k_senior_norm, pharmacy_per_1k_senior_norm)
Score_교통 = mean(lowfloor_bus_density_norm, subway_elevator_ratio_norm)
Score_복지 = mean(welfare_capacity_ratio_norm, (1 - solo_senior_ratio_norm))
Score_인프라 = mean(market_density_norm)
Score_안전 = mean(cctv_density_norm, (1 - senior_accident_rate_norm))
Score_기후 = mean(shelter_per_1k_senior_norm, park_area_ratio_norm)
```

### 4-3. 최종 종합 지수 (가중합산)

```
Total_Score = w₁·Score_의료 + w₂·Score_교통 + w₃·Score_복지 + w₄·Score_인프라
            + w₅·Score_안전 + w₆·Score_기후
```

**가중치 설정 방안 — 2가지 제시:**

| 방안                    | 방법                                                      | 장점                               | 단점                        |
| ----------------------- | --------------------------------------------------------- | ---------------------------------- | --------------------------- |
| A. 균등 가중            | 모든 w = 1/6 ≈ 0.167                                      | 단순, 투명, 편향 없음              | 영역 간 중요도 차이 무시    |
| B. AHP 기반 전문가 가중 | 선행연구·정책 문헌에서 도출한 쌍대비교 행렬로 가중치 산출 | 정책적 근거 확보, 심사위원 설득력↑ | 근거 문헌 필요, 주관성 잔존 |

**권고: 방안 A(균등 가중)를 기본으로 채택하되, 발표 시 방안 B 감수성 분석(sensitivity analysis) 결과도 함께 제시.** "의료 가중치를 2배로 높이면 순위가 어떻게 바뀌는가?"를 보여주면 창의성 + 정책 기여도 점수 모두 상승.

### 4-4. 순위 산정 및 등급 분류

- 행정동 467개를 Total_Score 기준 내림차순 정렬.
- **5등급 분류** (자연적 분류: Jenks Natural Breaks 또는 분위수 기반):
  - A등급 (상위 20%): 노인 친화 우수 구역
  - B등급 (20~40%): 양호
  - C등급 (40~60%): 보통
  - D등급 (60~80%): 주의 필요
  - E등급 (하위 20%): 노인 소외 취약 구역

## 5. 시각화 구현 방안 및 정책 인사이트 도출

### 5-1. 시각화 구성 (대시보드 레이아웃)

2025 우승작의 구성(WHY → CONTENTS → CONCLUSION)을 참고하여, PPT/PDF 발표자료 + 별도 시각화 결과물로 구성.

**발표 스토리 흐름:**

```
[1] 왜 노인인가? (WHY)
    → 초고령사회 진입 + 보행일상권 맥락 제시

[2] 서울 노인의 현재 (OVERVIEW)
    → 자치구별 고령화율 코로플레스 + 독거노인 비율 히트맵

[3] 7대 영역 분석 (DEEP DIVE)
    → 영역별 히트맵 스몰 멀티플 (Small Multiples)
    → 각 영역에서 TOP 5 / BOTTOM 5 행정동 비교

[4] 종합 지수 (SYNTHESIS)
    → 최종 노인친화지수 코로플레스 맵
    → 자치구 단위 레이더 차트 (7축)
    → 2×2 매트릭스: "시설은 많지만 노인은 더 많은 동" vs "시설도 적고 노인도 적은 동"

[5] 취약 유형화 (PROFILING)
    → K-means 클러스터링으로 3~5개 유형 도출
    → 유형별 레이더 차트 프로필
    → "의료 취약형", "교통 고립형", "복합 소외형" 등 네이밍

[6] 정책 제안 (CONCLUSION)
    → 유형별 맞춤 정책 제안 + 우선순위 매트릭스
```

### 5-2. 개별 시각화 차트 유형

### 5-3. 정책 인사이트 시나리오 (5가지)

**시나리오 1: "의료 사각지대" 식별 → 1차 진료기관 유치 정책**

> 분석 결과 강서구 A동은 65세+ 인구 비율이 25%로 서울 상위 10%이나, 노인 1,000명당 1차 진료기관 수는 0.8개로 서울 평균(2.3개)의 1/3 수준. 의료 접근성 점수가 0.12로 최하위권. → **정책 제안**: 해당 동에 공공 의료기관(보건분소) 설치 또는 이동진료 차량 배치 우선 대상으로 선정.

**시나리오 2: "교통 고립형" 구역 → 저상버스 노선 재편**

> 노원구 B동은 종합지수 D등급이나, 세부 프로필을 보면 복지(0.7)·의료(0.6) 점수는 양호한 반면 교통 점수가 0.08로 압도적 최하위. 저상버스 정류장이 행정동 내 0개, 지하철역 엘리베이터도 미설치. → **정책 제안**: 해당 동을 경유하는 기존 버스 노선에 저상버스 1대 추가 배치 시, 교통 점수가 0.08 → 0.35로 상승하여 종합 등급이 D → C로 개선 가능.

**시나리오 3: "복합 소외형" 밀집 권역 → 권역별 통합 지원 센터**

> 강북 동북권(도봉·강북·노원 일부)에 E등급 행정동이 지리적으로 군집. 의료·교통·복지 모두 하위 30%. → **정책 제안**: 개별 시설 확충보다 "원스톱 노인 생활지원센터"를 권역 거점에 설치하여, 1차 의료 + 복지 상담 + 디지털 교육을 통합 제공하는 것이 비용 효율적.

**시나리오 4: "기후 취약형" 구역 → 무더위쉼터 증설 우선순위**

> 관악구 C동은 65세+ 독거노인 비율이 38%로 서울 최상위이나, 무더위쉼터가 행정동 내 1개소에 불과. 폭염 시 실질적 대피 접근성이 극히 낮음. → **정책 제안**: 경로당·주민센터 등 기존 공공시설을 무더위쉼터로 추가 지정하여 접근 거리 단축.

**시나리오 5: 수요-공급 갭 분석 → "착시 양호 구역" 경고**

> 종로구 D동은 종합지수 B등급(양호)이나, 이는 문화재·관광 인프라에 의한 시설 수 집계 왜곡. 실제 노인 수요(독거노인 비율 30%) 대비 실질 복지시설 정원은 부족. → **정책 제안**: 종합지수만이 아닌 "수요 보정 지수"를 병행하여 정책 우선순위 재조정 필요.
