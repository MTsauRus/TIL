## 1. 프로젝트 정체성

### 1-1. 한 줄 정의

> **2040 서울도시기본계획이 약속한 "보행 30분 자족 생활권"은 표준 시민의 시간이다. 노인의 보행속도로 환산하면 그 약속의 도달 영역은 크게 달라진다. 본 프로젝트는 "노인이 걸어서 닿는 서울"을 5개 생활 도메인에서 재측정하여, 보행일상권 재정비 시 참고할 진단 자료를 제공한다.**

### 1-2. 문제 의식

2040 서울도시기본계획(p.46)은 "도보 30분 자족 생활권"을 핵심 비전으로 제시한다. 그러나 이 30분은 **표준 시민의 보행속도(약 4km/h, 1.1m/s)** 를 가정한다.

같은 계획서(p.27)는 서울 인구의 19%(2025)가 65세 이상이며 2040년에는 **약 32%** 에 이를 것이라고 명시한다. 75세 이상 인구는 2010년 이후 **연평균 6.8%** 씩 증가 중이다.

**약속된 시간과 그 약속을 받을 시민의 시간이 일치하지 않는다.**

| 차원 | 표준 시민 | 65~74세 | 75~84세 | 85세+ |
|------|-----------|---------|---------|-------|
| 보행속도 (m/s) | 1.1 ~ 1.4 | 0.95 | 0.78 | 0.58 |
| 도보 30분 도달 거리 (m) | 약 2,000 | 약 1,710 | 약 1,400 | 약 1,040 |
| 1 km 도달 시간 (분) | 12.5 | 17.5 | 21.4 | 28.7 |

→ 계획의 30분 약속은 **65~74세에게는 약 35분, 75세 이상에게는 약 45분, 85세 이상에게는 약 60분**.

### 1-3. 프로젝트 가치 (정책 기여도의 핵심 어휘)

본 진단은 2040 계획을 **부정하지 않는다**. 방향성은 옳다.

다만 그 계획이 **유니버설 디자인(Universal Design)** 을 획득하려면 표준 보행속도 기준의 재검토가 필요하다. 본 프로젝트는 그 재검토의 **기초 입력 자료(reference material)** 를 제공한다. 최종 판단은 정책담당자와 지역 전문가의 몫이다.

## 2. 공통 철학과 방법론

### 2-1. 분석 초점 (무엇을 다루고, 무엇을 다루지 않는가)

**포함** — 노인이 **걸어서** 닿아야 하는 것들:
- 생활 필수시설 (1차 의료·약국·시장·은행·주민센터·복지관·경로당)
- 무장애 교통 접점 (저상버스 정류장·지하철 엘리베이터)
- 위기 시각 대피시설 (무더위쉼터·한파쉼터)
- 사회적·여가 접점 (공원·종교시설)

**제외** — 명시적으로 **다루지 않는** 것:
- **응급의료 접근성·골든타임** — 응급차가 찾아오는 관점이지 노인이 걸어가는 관점이 아님
- **24시간 시설 운영시간 분석** — 폭염 정오에 쉼터 열린다는 건 당연함. 허구의 문제의식
- **시설 수 단순 순위 매기기** — 도달 가능성으로 측정, 순위가 아닌 지문으로 시각화

### 2-2. 핵심 가설 (전 도메인 공통)

> **H**: 행정동 단위 시설 수가 동일하더라도, 노인 보행속도 기준 30분 도달 가능 영역과 그 안의 시설 수는 청년 기준 대비 유의미하게 줄어든다. 줄어드는 정도(격차)는 동별로 차이가 크며, 그 격차에 거주하는 노인 인구의 분포는 행정 경계와 일치하지 않는다.

각 도메인은 이 가설을 자기 분야에서 검증한다.

### 2-3. 출발점 방법론 (매우 중요 — 모든 도메인 공통)

보행 접근성 분석의 **출발점**을 어떻게 정의하느냐가 결과 신뢰도를 좌우한다.

#### 잘못된 접근 (하지 말 것)
행정동 기하학적 중앙점(centroid) 1개에서 출발 → 등시선 1개 생성. 이유: 노인 거주 밀집 지역과 centroid 불일치, 산지·하천 포함 동은 무의미, 면적 큰 동에서 격차 왜곡.

#### v6 채택 방법: 500m 격자 기반 다중 출발점 + 인구 가중 집계

```
Step 1. 서울 전역에 500m × 500m 격자 생성 (약 1,500개 셀)
Step 2. 각 격자 셀에 65세+ 추정 인구 분배 (dasymetric 보간)
Step 3. 각 격자의 centroid를 개별 출발점으로 삼아 노인 보행 30분 등시선 계산
Step 4. 격자별로 30분권 내 도달 가능 시설 수 / 유형 다양성 산출
Step 5. 격자 결과를 행정동 경계로 집계하되, 격자 내 노인 인구를 가중치로 사용
     → "노인이 많이 사는 격자"의 결과가 동 집계에 더 큰 영향
```

이 방법론은 §5 "공통 인프라 (뼈대)"에서 김성령이 먼저 구현한다. 다른 팀원은 이 모듈을 **호출만** 하면 된다 (§3-3 참고).

### 2-4. 수요 가중 (인원 단위 결과 필수)

모든 도메인의 최종 결과는 **두 가지 단위로 표현**한다:

1. **비율/면적 격차** (예: 청년 대비 노인 30분권 면적 손실 비율)
2. **영향받는 노인 인원** (예: 격차 영역에 거주하는 65세+ 추정 인원)

인원 단위가 정책 판단의 실질적 근거가 된다. 비율만 보고하면 안 된다.

## 3. 도메인 표준 프로세스 (모든 팀원이 따를 공통 흐름)

각 도메인 담당자는 아래 7단계를 반복한다. 순서와 산출물을 맞춰야 §6 통합 시각화가 가능하다.

### Stage 1. 자기 도메인 시설 데이터 수집
- 데이터 카탈로그 §4에서 자기 도메인 필수 데이터 확인
- `data/raw/<domain_short>/` 디렉토리에 원본 CSV/API 다운로드 결과 보관
- 데이터 출처·버전·수집일을 `data/raw/<domain>/README.md`에 기록

### Stage 2. 행정동 매핑 및 정제
- 좌표(lat, lon) 데이터는 `src/common/geo_utils.py`의 `assign_admin_dong()` 호출
- 자치구 단위 데이터는 자치구 코드 기준 병합
- 결측치·중복 처리 후 `data/interim/<domain>_clean.csv` 저장

### Stage 3. 청년 vs 노인 30분권 도달 가능성 계산
**공통 모듈 사용**:
```python
from src.common.isochrone import dual_speed_coverage

# 자기 도메인 시설 좌표 넘기면 격자별 도달 가능 시설 수 반환
result = dual_speed_coverage(
    facilities=df_my_domain,  # lon, lat 컬럼 필수
    grid=grid_500m,            # 공통 인프라에서 제공
    speeds={'young': 1.20, 'senior': 0.78},
    time_min=30
)
# result: grid_id, young_count, senior_count, young_types, senior_types
```

### Stage 4. 파생 변수 계산 (도메인별로 §4에 명시)
- 격자별 `{domain}_gap_ratio` = 1 - senior_count / young_count
- 동별 집계: 격자 내 노인 인구(`pop_65plus`)를 가중치로 평균
- `{domain}_senior_affected`: 청년 30분권에는 포함되나 노인 30분권에는 없는 시설에 대응하는 추정 노인 인원

### Stage 5. 도메인 특화 시각화 2~3종 제작
각 도메인은 최소 다음 3종의 시각화 산출물을 만든다:
1. **청년·노인 30분권 오버레이 예시 (1개 동 줌인)** — 메인 차트
2. **서울 전체 도메인 격차 코로플레스** — 동별 분포
3. **도메인 특화 인사이트 차트** (§4에서 도메인별로 명시)

### Stage 6. 통합용 데이터 제출
김성령이 §6 종합 지문 시각화를 만들려면 각 도메인이 표준 형식의 CSV를 제출해야 한다:

```
data/processed/<domain>_summary.csv
컬럼:
- adm_dr_cd         : 행정동 코드 (8자리)
- adm_dr_nm         : 행정동명
- senior_count_30min: 동별 노인 가중 평균 접근 시설 수
- young_count_30min : 동별 청년 가중 평균 접근 시설 수
- gap_ratio         : 0~1 격차 비율
- types_count       : 시설 유형 다양성 (0~N)
- senior_affected   : 격차 영역에 거주하는 노인 추정 인원
```

이 형식을 지키면 김성령이 §6 지문 시각화를 자동 생성할 수 있다.

### Stage 7. PR 제출
- 자기 도메인의 코드·데이터·차트를 `branch: domain/<your_name>` 브랜치에 PR
- PR 설명에 (a) 무엇을 했는지, (b) 발견한 인사이트, (c) 다음 할 일

### D1. 교통 · 이동 접근성 · 노인 보행 안전 — 담당: 유호준

#### 주제 의식

노인이 동네 밖으로 이동하려면 **저상버스와 지하철 엘리베이터** 가 유일한 무장애 수단이다. 청년은 일반 버스·계단도 사용 가능하므로 이 차원에서 격차가 가장 극적으로 드러난다. 또한 동네 안에서 **걸어 다닐 때의 안전** — 노인 교통사고 다발 구간 — 도 "접근 가능성"의 일부다.

#### 필수 데이터

| # | 데이터 | ID | 출처 | 단위 | 우선순위 |
|---|--------|-----|------|------|---------|
| 1 | 서울시 버스 정류소 위치 | OA-1094 | 열린데이터광장 | 좌표 | ★★★ |
| 2 | 서울시 저상버스 도입 노선 및 보유율 | OA-22229 | 열린데이터광장 | 노선 | ★★★ |
| 3 | 서울시 저상버스 운행 노선수·대수 | OA-22221 | 열린데이터광장 | 자치구 | ★★ |
| 4 | 서울교통공사 엘리베이터 설치현황 | OA-13241 | 열린데이터광장 | 역사 | ★★★ |
| 5 | 서울교통공사 에스컬레이터 설치현황 | OA-13242 | 열린데이터광장 | 역사 | ★★ |
| 6 | 서울 지하철 역사 노약자 장애인 편의시설 | OA-11573 | 열린데이터광장 | 역사 | ★★★ |

| 8 | 도로교통공단 TAAS 노인 보행사고 좌표 | — | 공공데이터포탈 | 좌표 | ★★★ |

#### 도메인 특화 파생 변수

| 변수 | 산출식 | 의미 |
|------|--------|------|
| `lowfloor_bus_stops` | 저상버스 운행 노선 ∩ 버스 정류소 위치 | 동별 저상버스 정류장 수 |
| `subway_elevator_ratio` | 엘리베이터 보유 역 / 동 내 전체 역 | 무장애 지하철 접근 비율 |
| `barrier_free_transit_score` | 0.6 × lowfloor_bus + 0.4 × subway_elevator | 무장애 교통 통합 점수 |
| `senior_accident_per_10k` | 노인 보행사고 사상자 / (동별 65+ 인구 / 10,000) | 노인 만명당 보행사고율 |
| `accident_hotspot_in_dong` | 노인 보행사고 좌표 개수 (500m 그리드 내) | 사고 다발 그리드 식별 |

#### 도메인 특화 시각화 (필수 3종)

**D1-A: 저상버스 정류장 + 노인 15분 보행 등시선 오버레이 (1개 동 줌인)**
- 배경: 행정동 경계
- 점: 저상버스 정류장, 일반 정류장 (다른 색)
- 등시선: 저상버스 정류장 반경 15분(노인 보행속도 0.78 m/s 기준, 약 700m)
- 메시지: "이 동에서 노인이 걸어서 탈 수 있는 버스는 어디까지인가"

**D1-B: 서울 전체 무장애 교통 통합 점수 코로플레스**
- `barrier_free_transit_score` 동별 분포
- 5등급 자연 분류 (Jenks)

**D1-C: 노인 보행사고 다발 그리드 지도 + 골목/간선 분리**
- 500m 그리드별 노인 보행사고 사상자 수를 점밀도 또는 격자 색칠
- 사고 다발 그리드와 저상버스 미접근 그리드가 겹치는 영역 강조
- 메시지: "노인이 걷기에 위험한 구간 × 무장애 교통에서도 멀어진 구간" 이중 사각

#### 도메인 인사이트 목표

- 저상버스 정류장은 많지만 노인이 걸어서 닿을 수 없는 동 식별
- 노인 보행사고 다발 구간이 특정 시설 주변에 집중되는지 분석 (예: 전통시장 근처, 병원 근처)
- 자치구 평균에 가린 동별 격차 폭로

#### 구현 유의사항

- 저상버스 노선은 **전 노선**이 아니라 실제 운행 대수가 있는 노선만 필터링 (OA-22229의 `도입률` > 0 조건)
- 지하철역은 다수 행정동에 걸칠 수 있으므로 역사 1개를 인접 행정동 모두에 카운트하는 것이 아니라 역 출입구별 좌표로 분해하는 것을 권장 (OA-11573에 출입구 정보 포함 여부 확인)
- TAAS 노인 보행사고 좌표 데이터는 공공데이터포탈에서 신청·다운로드 필요. 발급 시간 있으니 W1 시작 즉시 신청.

### D2. 생활 인프라 접근성 — 담당: 심재현

#### 주제 의식

노인의 일상은 마트·전통시장·은행·주민센터에 크게 의존한다. 대형마트는 차로 가야 하지만 노인은 운전을 못 한다. 소량 구매가 가능한 **전통시장·중소 슈퍼** 가 핵심. 대면 금융 창구 — 은행 지점 — 도 디지털 소외된 노인에게 생존의 문제. 이 도메인은 노인이 걸어서 닿을 수 있는 "생활 거점"의 밀도와 다양성을 측정한다.

심재현은 Plotly 기반 대시보드 경험을 살려 §6 **통합 시각화 보조 역할** 도 함께 맡는다.

**D2-A: 전통시장 + 노인 30분 등시선 (1개 자치구 줌인, 예: 동작구)**
- 점: 전통시장, 슈퍼, 편의점, 은행 (다른 색·기호)
- 등시선: 각 전통시장 기준 노인 30분 반경
- 메시지: "노인이 걸어서 닿는 생활 거점의 실제 분포"

**D2-B: 서울 전체 생활 인프라 4유형 다양성 코로플레스**
- `daily_life_types` (0~4) 동별 범주형 컬러맵
- 4유형 모두 접근 가능한 동 vs 1유형만 접근 가능한 동의 공간 대비

- X축: 25개 자치구
- Y축: 동별 생활 인프라 접근성
- 각 자치구 내부의 동별 분산을 점으로 표시 → 자치구 평균은 양호하지만 일부 동이 사각인 사례 폭로

#### 도메인 인사이트 목표

- 전통시장 수 많은 자치구 ≠ 노인이 걸어서 닿는 자치구임을 보이기
- 은행 점포 통폐합 영향이 큰 지역 식별 (가능한 경우)
- 대형마트 중심 구역과 전통시장 중심 구역의 노인 접근성 대비

#### 통합 시각화 보조 역할 (심재현만 해당)

§6 김성령이 메인 통합 시각화를 담당하되, 심재현은 **Plotly 기반 인터랙티브 보조 대시보드 1종** 을 추가 제작한다.
- 동을 선택하면 5개 도메인의 격차를 radar chart로 보여주는 단순 위젯
- HTML 파일로 저장, 발표 시 현장에서 조작 시연 가능
- 단 이는 **보조** 이며, 메인 발표자료는 정적 PPT

- 다양성 부족한 동 식별

- 의료기관 총 수는 많지만 진료과목이 편중된 동 식별
- 보건소가 대체 가능한 1차 의료 부족 동 판별
- 노인 가중 기준의 "의료 사막" 동 TOP 20

#### 구현 유의사항

- 심평원 OpenAPI는 **신청 → 승인 시간** 있음. W1 시작 즉시 발급 신청.

- 진료과목 분류는 심평원의 `clCdNm` (분류명) 컬럼 활용.

- 대형 공원이 있다고 해서 노인이 걸어서 닿는 것은 아님을 시각적으로 증명

- 종교시설은 전국 데이터에서 서울만 필터링해야 함.

#### 도메인 특화 시각화 (필수 3종)

**D5-A: 위기 시각 듀얼 맵**
- 좌: 무더위쉼터 + 75+ 노인 15분 보행 등시선 + 독거노인 인구밀도 hexbin
- 우: 한파쉼터 같은 분석
- 두 위기 시각의 사각이 겹치는 영역 하이라이트

**D5-B: 동별 heat_risk_score TOP 20 막대 차트**
- 자치구 색상별 분류
- 동남권·서남권 상위 집중 예상 (2040 플랜 p.21 재해취약성 분석과 합치)

**D5-C: 폭염 시간대 노인 사망률 정점 차트 (의학적 근거)**
- 시계열: 시간(12-18시)
- Y축: 폭염 기간 노인 사망률 (통계청 연도별 평균)
- 정점 강조
- 메시지: "왜 노인 15분 보행 도달이 생명과 직결되는가"

#### 도메인 인사이트 목표

- 쉼터는 전 동에 있지만 75+ 독거노인에게는 거리상 사각인 동 식별
- 2040 플랜의 재해취약성 분석(자치구 단위)에 동 단위 정밀도 추가
- 폭염일수 + 독거노인 비율 + 보행 거리 3중 결합으로 우선순위 도출

## 7. 작업 원칙

### 7-1. 어휘 원칙 (매우 중요)

| 쓰지 말 것 | 대신 쓸 것 |
|----------|----------|
| 최적 입지 | 주목할 만한 후보 지점 |
| 여기에 지어야 한다 | 격차 완화 관점에서 검토할 여지 있음 |
| 처방 (Prescription) | 참고 자료 (Reference material) |
| 정책 제안 | 정책 결정의 입력값 |
| 취약 / 소외 (남발 금지) | 격차, 접근성 부족, 재정비 여지 |
| 당장 해결해야 한다 | 재정비 시 검토 가능한 |

**이유**: 심사위원과 정책담당자는 데이터 분석팀의 오만한 단정을 거부감으로 받아들인다. 겸손한 언어로 재료를 제공하는 것이 정책 기여도에서 훨씬 높게 평가된다.

### 7-2. 시각화 차별화 원칙

| ✅ 메인으로 사용 | ❌ 메인으로 사용 금지 |
|----------------|-------------------|
| 등시선(Isochrone) 오버레이 | 단순 코로플레스만 나열 |
| Slope chart (청년 vs 노인) | 2x2 산점도 메인 (2025 우승작 카피) |
| Radar small multiples (지문) | 4분면 A/B/C/D 라벨링 (2025 우승작 카피) |
| 격자 점밀도 + 등시선 중첩 | 단순 막대 차트만 |
| 의학적 근거 시계열 | — |

**코로플레스는 보조로만. 메인 시각화의 50% 이하**.

### 7-3. 데이터 정직성

- 모든 가정(보행속도, 시간 기준, 보간 방식)은 발표 시 한계로 명시
- 데이터 출처·버전·수집일을 `data/raw/<domain>/README.md`에 기록
- 결측이나 누락 차원은 "이 도메인은 데이터 접근성 한계로 보조 지표로만 활용"이라고 솔직히 표기

### 시각화 차별화
- [ ] **동네별 인프라 지문 Small Multiples** (통합-A)가 포함되었는가?
- [ ] 코로플레스 단독 비중 ≤ 40%인가?
- [ ] isochrone 오버레이·slope chart·radar small multiples 중 2개 이상 포함?
- [ ] 2x2 매트릭스가 메인 시각화에 **없는가**?
