# 공지

반갑습니다. 스터디장을 맡아버린 김성령입니다.. 하기로 했으니 최소한의 형식 및 규칙 공지드릴게요.

### 이 스터디가 어떻게 돌아갈 예정인가

- 2일에 한 번씩 문제를 공지드릴 예정입니다.
- 월화 / 수목 / 금토일. 이렇게, 주 3문제 풀이가 목표입니다.

### 무슨 문제가 출제되나요

- codetree의 삼성 / 현대 기출문제를 풀 예정입니다.
- 메인 1문제 / 서브 1문제로 진행합니다.
- 서브를 드리는 이유는 다음과 같습니다.
  - 본인이 너무 성실한 나머지 예전에 메인 1문제를 푼 적이 있는 경우.
  - 본인이 너무 근면한 나머지 한 문제를 더 풀고 싶은 경우.
- 규칙은 2일 내에 메인 한 문제만 풀면 됩니다.

### 활용 메신저

- 오직 MM을 사용할 예정입니다.
- 더 좋은 의견 있다면 받겠습니다.

### 인증 양식

- 이 방에 어떤 방식이든 자신의 풀이를 인증하면 됩니다. 예시 사진처럼요.
- 다음의 양식을 따르겠습니다.

```md
푼 문제: [문제이름]
체감 난이도: [대충 난이도 평가]
핵심 접근법: [내가 생각하기에 멋있게 푼 것 같은 코드 부분 공유]
```

- 예시는 아래와 같습니다.

푼 문제: 아기 고래의 첫 항해

체감 난이도: 평이했다.

핵심 접근법: dr, dc 배열 이외의 dd 배열을 도입해서 코드를 간결하게 짰다.

```java
    int[] dr = {1, -1, 0, 0};
    int[] dc = {0, 0, 1, -1};
    int[] dd = {0, -1, 1, 2};

    ~~ 이하 코드 생략 ~~
```
